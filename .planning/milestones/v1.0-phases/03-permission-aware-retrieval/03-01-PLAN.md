  <action>
    Create PermissionFilterService to encapsulate permission filtering logic:
    
    1. Create service class at `src/Service/PermissionFilterService.php`
    2. Register service in `social_ai_indexing.services.yml`:
       - Service name: social_ai_indexing.permission_filter
       - Arguments: @group.membership_loader, @current_user, @current_route_match
    3. Implement methods:
       
       a) `getAccessibleGroupIds(AccountInterface $account): array`
          - Use group.membership_loader->loadByUser($account)
          - Extract Group IDs from memberships
          - Return unique array of integer Group IDs
          - Return empty array for anonymous users (no group memberships)
       
       b) `isCommunityWideQuery(): bool`
          - Check current route for group context
          - Use current_route_match->getParameter('group')
          - Return TRUE if no group in route (community-wide)
          - Return FALSE if group context exists (group-scoped)
       
       c) `applyPermissionFilters(QueryInterface $query, AccountInterface $account, ?int $scopeGroupId = NULL): void`
          - If anonymous: add condition content_visibility = 'public' only
          - If community-wide: add condition content_visibility IN ['public', 'community']
          - If group-scoped with scopeGroupId: add condition group_id = scopeGroupId
          - If group-scoped without specific group: add condition group_id IN [user's accessible groups]
          - If user has no accessible groups and not community-wide: add impossible condition (group_id = -1)
    
    4. Use dependency injection for all services
    5. Add proper type hints and PHPDoc
    
    DO NOT:
    - Do not cache permission decisions (permissions can change at any time)
    - Do not bypass group.membership_loader with raw SQL queries
    - Do not assume all content has visibility field (check before filtering)
  </action>
  <verify>
    <automated>ddev drush php-eval "print_r(\Drupal::service('social_ai_indexing.permission_filter')->getAccessibleGroupIds(\Drupal::currentUser()));"</automated>
    <manual>Verify service is registered and returns array of Group IDs for logged-in user</manual>
  </verify>
