<task type="auto">
  <name>Task 2: Create CitationMetadata processor for citation URLs</name>
  <files>html/modules/custom/social_ai_indexing/src/Plugin/search_api/processor/CitationMetadata.php</files>
  <action>
Create a Search API processor that adds citation metadata (URL, title, type) to indexed items for use in AI responses.

Follow the pattern from GroupMetadata.php and ContentVisibility.php processors:
1. Extend ProcessorPluginBase
2. Define properties: citation_url, citation_title, citation_type
3. Implement addFieldValues() to:
   - Get the entity from the item
   - Generate canonical URL using $entity->toUrl('canonical', ['absolute' => TRUE])
   - Add title via $entity->label()
   - Add type via $entity->bundle() or $entity->getEntityTypeId()

Key implementation details:
- Use $item->getOriginalObject()->getValue() to get the entity
- Handle both nodes and comments (comments should link to parent entity)
- Wrap URL generation in try/catch to handle edge cases
- Never let the processor fail silently - log errors but continue

Do NOT use \Drupal::url() - use the entity's toUrl() method for proper URL generation.
  </action>
  <verify>
    <automated>ddev drush php-eval "echo class_exists('Drupal\social_ai_indexing\Plugin\search_api\processor\CitationMetadata') ? 'EXISTS' : 'MISSING';"</automated>

<task type="auto">
  <name>Task 3: Create AI Assistant configuration with RAG</name>
  <files>config/sync/ai_assistant.ai_assistant.group_assistant.yml</files>
  <action>
Create an AI Assistant entity configuration with RAG enabled for the Group knowledge assistant.

Create config file at html/modules/custom/social_ai_indexing/config/optional/ai_assistant.ai_assistant.group_assistant.yml:

```yaml
langcode: en
status: true
dependencies:
  module:
    - ai_assistant_api
id: group_assistant
label: 'Group Knowledge Assistant'
description: 'Answers questions about Group content using RAG retrieval'
system_prompt: |
  You are a helpful assistant that answers questions about Group content.
  
  RULES:
  1. Use ONLY the provided context to answer questions.
  2. If the context doesn't contain relevant information, say: "I couldn't find information about that in the Group content."
  3. Always cite your sources using markdown links: [Source: Title](url)
  4. Be concise but thorough.
  5. Never make up information or URLs.
chat_model: 'deepseek__deepseek-chat'
fallback_model: ''
stream_response: true
rag_enabled: true
rag_database: 'social_posts'
rag_threshold: 0.7
rag_max_results: 5
```

After creating the config:
1. Run `ddev drush cim -y --partial --source=html/modules/custom/social_ai_indexing/config/optional`
2. Verify with `ddev drush entity:list ai_assistant`
3. Copy to config/sync/ai_assistant.ai_assistant.group_assistant.yml for deployment

Do NOT use a higher rag_threshold than 0.7 - this would cause "I couldn't find" responses when content exists.
Do NOT set rag_max_results higher than 5 - this would increase latency beyond 10 seconds.
  </action>
