    **CRITICAL: Correct base class and plugin ID mapping:**

    | Class | Plugin ID | Base Class | Source YAML | Entity Type |
    |-------|-----------|------------|-------------|-------------|
    | LocalnodesUserTerms | localnodes_user_terms | `\Drupal\social_demo\Plugin\DemoContent\UserTerms` | content/entity/user-terms.yml | taxonomy_term |
    | LocalnodesFile | localnodes_file | `\Drupal\social_demo\DemoFile` | content/entity/file.yml | file |

    | LocalnodesGroup | localnodes_group | `\Drupal\social_demo\DemoGroup` | content/entity/group.yml | group |
    | LocalnodesEventType | localnodes_event_type | `\Drupal\social_demo\Plugin\DemoContent\EventType` | content/entity/event-type.yml | taxonomy_term |
    | LocalnodesEvent | localnodes_event | `\Drupal\social_demo\Plugin\DemoContent\Event` | content/entity/event.yml | node |
    | LocalnodesTopic | localnodes_topic | `\Drupal\social_demo\Plugin\DemoContent\Topic` | content/entity/topic.yml | node |
    | LocalnodesPost | localnodes_post | `\Drupal\social_demo\Plugin\DemoContent\Post` | content/entity/post.yml | post |
    | LocalnodesComment | localnodes_comment | `\Drupal\social_demo\DemoComment` | content/entity/comment.yml | comment |
    | LocalnodesLike | localnodes_like | `\Drupal\social_demo\Plugin\DemoContent\Like` | content/entity/like.yml | vote |
    | LocalnodesEventEnrollment | localnodes_event_enrollment | `\Drupal\social_demo\Plugin\DemoContent\EventEnrollment` | content/entity/event-enrollment.yml | event_enrollment |

    **IMPORTANT notes on base classes:**
    - For Topic: Extend `\Drupal\social_demo\Plugin\DemoContent\Topic` (NOT DemoNode directly) because Topic has critical `prepareTopicType()` and `prepareAttachment()` methods.
    - For Event: Extend `\Drupal\social_demo\Plugin\DemoContent\Event` (NOT DemoNode directly) because Event has `createEventDate()`, `prepareEventType()`, and `prepareEventManagers()` methods.
    - For Post: Extend `\Drupal\social_demo\Plugin\DemoContent\Post` (NOT DemoEntity directly) because Post has `createDate()` and handles images/recipients/groups.
    - For Like: Extend `\Drupal\social_demo\Plugin\DemoContent\Like` (NOT DemoEntity directly) because Like has custom `getEntry()` for vote entities.
    - For EventEnrollment: Extend `\Drupal\social_demo\Plugin\DemoContent\EventEnrollment` (NOT DemoEntity directly) because it has custom `getEntry()` for enrollment fields.
    - For EventType: Extend `\Drupal\social_demo\Plugin\DemoContent\EventType` because it handles taxonomy terms with specific vocabulary assignment.
    - For UserTerms: Extend `\Drupal\social_demo\Plugin\DemoContent\UserTerms` because it handles profile_tag vocabulary terms.
    - For File, User, Group, Comment: Use the DemoFile, DemoUser, DemoGroup, DemoComment base classes respectively.

    **Pattern for each plugin (example for LocalnodesUser):**
    ```php
    <?php

    namespace Drupal\localnodes_demo\Plugin\DemoContent;

    use Drupal\social_demo\DemoUser;

    /**
     * LocalNodes User Plugin for demo content.
     *
     * @DemoContent(
     *   id = "localnodes_user",
     *   label = @Translation("LocalNodes User"),
     *   source = "content/entity/user.yml",
     *   entity_type = "user"
     * )
     */
    class LocalnodesUser extends DemoUser {

    }
    ```

    All classes follow this exact pattern -- the only differences are: namespace, use statement, class name, annotation values (id, label, source, entity_type), and parent class.
