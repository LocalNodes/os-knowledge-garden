**Step 1: Verify the LocalNodes/os-knowledge-garden remote is configured.**
Check `git remote -v` for a remote pointing to `github.com/LocalNodes/os-knowledge-garden`. If missing, add it:
```bash
git remote add upstream git@github.com:LocalNodes/os-knowledge-garden.git
```
(Or whatever remote name is appropriate — the key is that pushing to this remote triggers the GitHub Actions workflow.)

**Step 2: Push Docker/CI files to GitHub.**
Ensure all files from Plan 09-01 are committed and pushed to the `main` branch of `LocalNodes/os-knowledge-garden`:
```bash
git push upstream main
```
(If the remote is already `origin` pointing to LocalNodes/os-knowledge-garden, use `git push origin main`.)

**Step 3: Verify CI workflow triggered.**
Check GitHub Actions status:
```bash
gh run list --repo LocalNodes/os-knowledge-garden --limit 3
```
Wait for the build to complete. The workflow should produce `ghcr.io/localnodes/os-knowledge-garden:latest`.

**Step 4: Verify image exists in GHCR.**
```bash
gh api /orgs/LocalNodes/packages/container/os-knowledge-garden/versions --jq '.[0].metadata.container.tags'
```
Should show `["latest", "<sha>"]`.

If the CI fails, check the logs:
```bash
gh run view --repo LocalNodes/os-knowledge-garden --log-failed
```
Common failures: composer patch application errors (check patch file paths match), PHP extension install failures (check Dockerfile apt-get packages).

**Step 1: Delete old Coolify apps (if they still exist).**
Use the Coolify MCP tools to delete the old apps that used the abandoned deploy repo approach:

```
mcp__coolify__application(action="delete", uuid="j8osgcos8scos0ccw44wgo00", delete_volumes=true)
```

If either app no longer exists, the delete will fail gracefully — that's fine.

**Step 2: Create Cascadia app.**
Use `create_public` (not `create_github` — per Phase 09-02 decision, `create_github` has a Coolify API bug returning HTML):

```
mcp__coolify__application(
  action="create_public",
  project_uuid="n0sc0wcog8k4kkc48k0k0wo0",
  server_uuid="q4okokcg8occ88w00c4kg0sw",
  environment_name="production",
  git_repository="https://github.com/LocalNodes/os-knowledge-garden",
  git_branch="main",
  build_pack="dockercompose",
  name="cascadia",
  ports_exposes="80"
)
```

After creation, update the `docker_compose_location` to point to the correct file:
```
mcp__coolify__application(action="update", uuid="<cascadia-uuid>", docker_compose_location="/docker-compose.yml")
```
Note: Capture the UUID returned from the create call.

| DB_PASSWORD | (generate random: `openssl rand -hex 16`) |
| DB_ROOT_PASSWORD | (generate random: `openssl rand -hex 16`) |
| DRUPAL_HASH_SALT | (generate random: `openssl rand -hex 32`) |
| DRUPAL_ADMIN_PASS | localnodes2026 |
| SERVICE_FQDN_OPENSOCIAL | cascadia.localnodes.xyz |
| SERVICE_URL_OPENSOCIAL | https://cascadia.localnodes.xyz |

Use `mcp__coolify__env_vars(resource="application", action="create", uuid="<cascadia-uuid>", key="...", value="...")` for each.

**Step 6: Set domain mapping for both apps.**
Domain mapping in Coolify for docker-compose apps uses the `SERVICE_FQDN_OPENSOCIAL` env var. Coolify auto-generates Traefik labels from this. The env var set in Steps 3 and 5 should handle this. Verify by checking the app details.

**Step 7: Verify GEMINI_API_KEY is set before deploying.**
Before triggering deployment, confirm GEMINI_API_KEY is actually set on both apps. A missing key causes AI indexing to fail silently, and fixing it requires a full reinstall cycle (delete volumes + redeploy).

For each app UUID (Cascadia and Boulder):
```
mcp__coolify__env_vars(resource="application", action="list", uuid="<app-uuid>")
```
Scan the returned list for a `GEMINI_API_KEY` entry. If absent or empty:
- **STOP and prompt the user** to provide the key before proceeding
- Do NOT trigger deployment without a confirmed GEMINI_API_KEY

**Step 8: Deploy both apps.**
```
mcp__coolify__deploy(tag_or_uuid="<cascadia-uuid>", force=true)
mcp__coolify__deploy(tag_or_uuid="<boulder-uuid>", force=true)
```

**Step 9: Monitor deployment.**
Check deployment status:
```
mcp__coolify__deployment(action="list_for_app", uuid="<cascadia-uuid>")
mcp__coolify__deployment(action="list_for_app", uuid="<boulder-uuid>")
```

Initial deployment will take 4-8 minutes: Solr builds locally (~1 min), opensocial pulls from GHCR (~1 min), entrypoint runs site:install + demo content + indexing (~3-5 min).
