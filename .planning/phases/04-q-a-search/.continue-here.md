<root_cause_identified>
The RAG search tool returns "No results were found when searching in the rag index social_posts for the following prompt: LocalNodes." — confirmed via AI logging at /admin/config/ai/logging/collection/254.

The search works via direct drush query (returns 2 results), but fails through the chatbot path. This is because the permission-aware retrieval filter (built in Phase 3) restricts search results to groups the user is a member of. Admin user 1 is likely NOT a member of the demo groups created by localnodes_demo content.

Evidence:
- AI log 252: Model correctly calls ai_search_rag_search tool with args {"search_string": "LocalNodes"}
- AI log 254: Tool execution returns "No results were found" — model correctly follows instructions to say it couldn't find info
- Direct drush search (no permission filter): Returns results for "LocalNodes"
</root_cause_identified>
