The full code trace for the agent response path:
1. DeepChat sends POST to /api/deepchat with stream:false (agents don't stream, per isStreamingSupported())
2. DeepChatApi::api() → AiAssistantApiRunner::process() → AgentRunner::runAsAgent()
3. Agent runs RAG tool, generates response, returns ChatOutput with ChatMessage
4. Back in DeepChatApi: getNormalized() returns ChatMessage → createResponse()
5. createResponse() calls getCommonMarkConverter() → should return converter
6. $converter->convert($text) should convert markdown to HTML
7. Xss::filter() sanitizes (allows `<a>` with `href`)
8. Returns JSON { html: "...", should_continue: false }

All code analysis confirms the path SHOULD convert. The class IS installed. Yet the output is raw markdown. Next session should start by verifying CommonMark availability in the web context (not just CLI).
