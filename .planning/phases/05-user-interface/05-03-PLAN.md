must rely on DeepChat's built-in markdown rendering (which already makes links
blue and clickable) plus messageStyles for bubble-level distinction.

  <action>
Extend the existing `social_ai_indexing_preprocess_ai_deepchat()` function (added in Plan 05-01) to also configure DeepChat's `messageStyles` for visual distinction between AI and user messages.

CRITICAL: In the preprocess hook, `messageStyles` is a JSON-encoded string (just like `connect`). You must json_decode it, modify, and json_encode back. Do NOT treat it as a PHP array.

Add the messageStyles configuration in the SAME preprocess function, AFTER the existing group_id injection code:

```php
// Add this to the END of social_ai_indexing_preprocess_ai_deepchat(),
// AFTER the group_id injection block (outside the if ($group) block):

// Add visual distinction for AI vs user messages (UI-05).
// messageStyles is a JSON string at preprocess time.
$message_styles_json = $variables['deepchat_settings']['messageStyles'] ?? '{}';
$message_styles = is_string($message_styles_json) ? json_decode($message_styles_json, TRUE) : [];
if (!is_array($message_styles)) {
  $message_styles = [];
}
$message_styles['default']['ai']['bubble']['borderLeft'] = '3px solid #4a90d9';
$message_styles['default']['ai']['bubble']['backgroundColor'] = '#f8f9fa';
$variables['deepchat_settings']['messageStyles'] = json_encode($message_styles);
```

Also add a new `hook_page_attachments` implementation to ensure the AI distinction CSS loads on all pages where the DeepChat chatbot and Related Content block appear:

```php
/**

  $attachments['#attached']['library'][] = 'social_ai_indexing/ai-search';
}
```

Do NOT modify the existing group_id injection logic from Plan 05-01.
Do NOT add inline styles — use the messageStyles JSON configuration which DeepChat applies natively.
Do NOT add a separate library for just the CSS — reuse the ai-search library (the JS is harmless on non-search pages since it only attaches to .ai-search-form).
