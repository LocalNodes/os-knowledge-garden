Extend the existing `social_ai_indexing_deepchat_settings()` function (added in Plan 05-01) to also configure DeepChat's `messageStyles` for visual distinction between AI and user messages.

Add the messageStyles configuration AFTER the existing group_id injection code in the same function:

```php
// In social_ai_indexing_deepchat_settings(), after the group_id injection block:

// Add visual distinction for AI vs user messages (UI-05).
$message_styles = [];
if (isset($deepchat_settings['messageStyles']) && is_array($deepchat_settings['messageStyles'])) {
  $message_styles = $deepchat_settings['messageStyles'];
}
$message_styles['default']['ai']['bubble']['borderLeft'] = '3px solid #4a90d9';
$message_styles['default']['ai']['bubble']['backgroundColor'] = '#f8f9fa';
$deepchat_settings['messageStyles'] = $message_styles;
```

Also attach the ai-search library to the DeepChat block for citation styling:

```php
// Note: The library attachment cannot be done via hook_deepchat_settings
// since that hook only alters the DeepChat element attributes, not the
// Drupal render array. Citation styling in the CSS file targets
// .deepchat-element which is a class set by DeepChatFormBlock. The CSS
// is loaded globally via the library when the search page is accessed.
// For non-search pages, we need to attach the CSS via hook_page_attachments.
```

Add a new `hook_page_attachments` implementation to ensure the citation and AI distinction CSS loads on all pages where the DeepChat chatbot appears:

```php
/**
 * Implements hook_page_attachments().
 *
 * Attaches AI search CSS globally for citation link styling in the
 * DeepChat chatbot and AI content visual distinction.
 */
function social_ai_indexing_page_attachments(array &$attachments) {
  $attachments['#attached']['library'][] = 'social_ai_indexing/ai-search';
}
```

Do NOT modify the existing group_id injection logic from Plan 05-01.
