  <action>
    Create the localnodes_demo module directory structure at html/modules/custom/localnodes_demo/.

    1. Create localnodes_demo.info.yml:
       ```yaml
       name: 'LocalNodes Demo Content'
       type: module
       description: 'Provides bioregionalism-themed demo content for the LocalNodes.xyz community platform.'
       core_version_requirement: ^9 || ^10
       package: LocalNodes
       dependencies:
         - social_demo:social_demo
       ```

    2. Create 12 plugin classes in src/Plugin/DemoContent/. Each plugin follows the same pattern: a thin class with a @DemoContent annotation that specifies a unique plugin ID (prefixed with "localnodes_"), a label, the YAML source file path, and the entity type. The class extends the appropriate social_demo base class.

    **CRITICAL: Plugin class inheritance must match what social_demo uses:**
    - LocalnodesUserTerms extends DemoTaxonomyTerm (like social_demo UserTerms)
    - LocalnodesFile extends DemoFile (like social_demo File)
    - LocalnodesUser extends DemoUser (like social_demo User)
    - LocalnodesGroup extends DemoGroup (like social_demo Group)
    - LocalnodesEventType extends DemoTaxonomyTerm with social_event_type check (like social_demo EventType)
    - LocalnodesEvent extends \Drupal\social_demo\Plugin\DemoContent\Event (MUST extend the concrete Event class, not DemoNode, because Event has custom getEntry() with event-specific fields like field_event_date, field_event_address, field_event_type handling)
    - LocalnodesTopic extends \Drupal\social_demo\Plugin\DemoContent\Topic (MUST extend the concrete Topic class, not DemoNode, because Topic has custom getEntry() with topic-specific fields like field_topic_type, field_content_visibility, image handling)
    - LocalnodesPost extends \Drupal\social_demo\Plugin\DemoContent\Post (MUST extend the concrete Post class, not DemoEntity, because Post has custom getEntry() with post-specific fields like field_visibility, field_post, recipient, group, field_post_image handling)
    - LocalnodesComment extends DemoComment (like social_demo Comment)
    - LocalnodesLike extends \Drupal\social_demo\Plugin\DemoContent\Like (MUST extend the concrete Like class, not DemoEntity, because Like has custom getEntry() with vote-specific fields)
    - LocalnodesEventEnrollment extends \Drupal\social_demo\Plugin\DemoContent\EventEnrollment (MUST extend the concrete EventEnrollment class, not DemoEntity, because EventEnrollment has custom getEntry() with enrollment-specific fields)

    Plugin IDs and source files:
    - localnodes_user_terms -> content/entity/user-terms.yml -> entity_type: taxonomy_term
    - localnodes_file -> content/entity/file.yml -> entity_type: file
    - localnodes_user -> content/entity/user.yml -> entity_type: user
    - localnodes_group -> content/entity/group.yml -> entity_type: group
    - localnodes_event_type -> content/entity/event-type.yml -> entity_type: taxonomy_term
    - localnodes_event -> content/entity/event.yml -> entity_type: node
    - localnodes_topic -> content/entity/topic.yml -> entity_type: node
    - localnodes_post -> content/entity/post.yml -> entity_type: post
    - localnodes_comment -> content/entity/comment.yml -> entity_type: comment
    - localnodes_like -> content/entity/like.yml -> entity_type: vote
    - localnodes_event_enrollment -> content/entity/event-enrollment.yml -> entity_type: event_enrollment

    **IMPORTANT:** All plugin classes should be empty body (just extend the parent) EXCEPT LocalnodesEventType which needs the social_event_type module check, identical to the social_demo EventType class.

    3. Create empty content/entity/ directory with a .gitkeep file (YAML files will be created in Plan 02).

  <action>
    Download nature/bioregionalism-themed images from Unsplash for use as demo content images. Per user decision: "Use Unsplash for images -- All demo images (user avatars, group banners, post images, event images) should use Unsplash photos."

    Use curl to download images from Unsplash Source API (https://source.unsplash.com/) or direct Unsplash photo URLs. Save them as JPEG files in html/modules/custom/localnodes_demo/content/files/.

    **Image requirements:**

    Profile photos (12 images, ~400x400):
    - Download 12 portrait/headshot-style photos for user avatars
    - Name them: avatar-01.jpg through avatar-12.jpg
    - Use search terms like: portrait, headshot, person, face
    - These should be diverse people photos

    Content images (12 images, ~960x640 landscape):
    - Download 12 nature/landscape photos themed for bioregionalism
    - Name them: content-01.jpg through content-12.jpg
    - Use search terms like: watershed, forest, garden, permaculture, community-garden, river, landscape, mountain, farm, solar-panels, coastline, meadow
    - These will be used for group banners, topic images, event images

    **Download approach:**
    Use Unsplash direct URLs for specific curated photos. For each image:
    ```bash
    curl -L -o content/files/avatar-01.jpg "https://images.unsplash.com/photo-{PHOTO_ID}?w=400&h=400&fit=crop&crop=face"
    curl -L -o content/files/content-01.jpg "https://images.unsplash.com/photo-{PHOTO_ID}?w=960&h=640&fit=crop"
    ```

    If Unsplash URLs fail or rate-limit, create solid-color placeholder JPEGs using ImageMagick (available in DDEV):
    ```bash
    ddev exec convert -size 400x400 xc:#4a7c59 /var/www/html/modules/custom/localnodes_demo/content/files/avatar-01.jpg
    ddev exec convert -size 960x640 xc:#2d5a3d /var/www/html/modules/custom/localnodes_demo/content/files/content-01.jpg
    ```

    The exact approach (Unsplash vs placeholder) depends on network availability. Either way, 24 JPEG files must exist in content/files/ when done.

    **DO NOT reuse social_demo's generic stock photos** (per user decision).
  </action>
