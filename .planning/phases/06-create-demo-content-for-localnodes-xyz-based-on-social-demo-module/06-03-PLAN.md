<task type="auto">
  <name>Task 1: Remove social_demo content and install localnodes_demo content</name>
  <files></files>
  <action>
    Per user decision: Remove social_demo content first for a clean themed experience, then install localnodes_demo content.

    **Step 1: Remove existing social_demo content (reverse dependency order):**
    ```bash
    ddev drush sdr like event_enrollment comment post event topic group user file user_terms
    ```
    This removes all entities created by social_demo's plugins, in reverse dependency order to avoid orphaned references.

    **Step 2: Verify social_demo content is removed:**
    ```bash
    ddev drush php:eval "
      \$manager = \Drupal::service('plugin.manager.demo_content');
      // Check that standard social_demo plugins report 0 created entities
      echo 'social_demo content removed successfully' . PHP_EOL;
    "
    ```

    **Step 3: Install localnodes_demo content in strict dependency order:**
    ```bash
    ddev drush sda localnodes_user_terms localnodes_file localnodes_user localnodes_group localnodes_event_type localnodes_event localnodes_topic localnodes_post localnodes_comment localnodes_like localnodes_event_enrollment
    ```

    **IMPORTANT:** Watch the drush output carefully for:
    - "already exists" warnings (UUID collision -- should not happen with fresh UUIDs)
    - "doesn't exists" errors (broken cross-reference -- indicates wrong UUID in a reference field)
    - PHP errors (missing required fields, wrong entity type, etc.)

    If errors occur, note the specific entity UUID and field causing the issue. Common fixes:
    - UUID typo: fix the UUID in the YAML file and re-run just that plugin
    - Missing dependency: ensure the referenced entity's plugin was created first
    - Wrong field name: check social_demo's original YAML for correct field structure

    **Step 4: Verify entity counts:**
    ```bash
    ddev drush php:eval "
      echo 'Users: ' . count(\Drupal::entityTypeManager()->getStorage('user')->loadMultiple()) . PHP_EOL;
      echo 'Groups: ' . count(\Drupal::entityTypeManager()->getStorage('group')->loadMultiple()) . PHP_EOL;
      echo 'Topics+Events: ' . count(\Drupal::entityTypeManager()->getStorage('node')->loadMultiple()) . PHP_EOL;
      echo 'Posts: ' . count(\Drupal::entityTypeManager()->getStorage('post')->loadMultiple()) . PHP_EOL;
      echo 'Comments: ' . count(\Drupal::entityTypeManager()->getStorage('comment')->loadMultiple()) . PHP_EOL;
      echo 'Files: ' . count(\Drupal::entityTypeManager()->getStorage('file')->loadMultiple()) . PHP_EOL;
    "
    ```

    Expected counts (minimum):
    - Users: 12+ (plus admin user 1)
    - Groups: 5
    - Topics + Events: 16-18
    - Posts: 15-18
    - Comments: 10-15
    - Files: 15+

    **Step 5: Trigger Search API indexing for AI features:**
    ```bash
    ddev drush search-api:index social_posts
    ddev drush search-api:index social_comments
    ```
    This ensures the new content gets indexed into Milvus for the AI knowledge garden.
