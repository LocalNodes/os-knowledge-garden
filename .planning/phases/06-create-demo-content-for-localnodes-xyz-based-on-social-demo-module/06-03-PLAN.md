<task type="auto">
  <name>Task 1: Remove social_demo content and install LocalNodes demo content</name>
  <files></files>
  <action>
    Per user decision: "Remove social_demo content -- The existing social_demo demo content must be removed/disabled before installing custom content."

    **Step 1: Remove existing social_demo content (in reverse dependency order):**
    ```bash
    ddev drush sdr like event_enrollment comment post event topic group user file user_terms event_type
    ```
    This uses the social_demo drush `social-demo:remove` (sdr) command to cleanly delete all social_demo-created entities.

    **Step 2: Create LocalNodes demo content in dependency order:**
    ```bash
    ddev drush sda localnodes_user_terms localnodes_file localnodes_user localnodes_group localnodes_event_type localnodes_event localnodes_topic localnodes_post localnodes_comment localnodes_like localnodes_event_enrollment
    ```
    This uses the social_demo drush `social-demo:add` (sda) command to create all localnodes_demo entities.

    **Step 3: Verify entity counts:**
    ```bash
    ddev drush eval "
      echo 'Users: ' . count(\Drupal::entityTypeManager()->getStorage('user')->loadByProperties(['status' => 1])) . PHP_EOL;
      echo 'Groups: ' . count(\Drupal::entityTypeManager()->getStorage('group')->loadMultiple()) . PHP_EOL;
      echo 'Topics: ' . count(\Drupal::entityTypeManager()->getStorage('node')->loadByProperties(['type' => 'topic'])) . PHP_EOL;
      echo 'Events: ' . count(\Drupal::entityTypeManager()->getStorage('node')->loadByProperties(['type' => 'event'])) . PHP_EOL;
      echo 'Posts: ' . count(\Drupal::entityTypeManager()->getStorage('post')->loadMultiple()) . PHP_EOL;
      echo 'Comments: ' . count(\Drupal::entityTypeManager()->getStorage('comment')->loadMultiple()) . PHP_EOL;
    "
    ```

    Expected counts (approximate): 12+ users, 5 groups, 8-10 topics, 6-8 events, 15-18 posts, 10-15 comments.

    **Step 4: Clear caches** to ensure UI reflects new content:
    ```bash
    ddev drush cr
    ```

    If any drush command shows errors about missing entities or UUID collisions, examine the error output. Common fixes:
    - "already exists" warnings: UUID collision -- regenerate the conflicting UUID in the YAML
    - "doesn't exists" errors: Missing dependency -- check creation order
    - PHP errors: Missing required field in YAML -- compare with social_demo's YAML structure
