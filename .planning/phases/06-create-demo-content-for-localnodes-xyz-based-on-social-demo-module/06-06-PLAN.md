<task type="auto">
  <name>Task 1: Remove existing demo content and install boulder_demo content</name>
  <files></files>
  <action>
    Remove any existing demo content first for a clean experience, then install boulder_demo content.

    **NOTE on separate instances:** The boulder_demo module is designed to run on a SEPARATE LocalNodes instance from localnodes_demo. However, on this development environment both modules may be enabled. We need to remove any previously active demo content (social_demo and/or localnodes_demo) before installing boulder_demo content, to avoid entity count confusion and duplicate content.

    **Step 1: Remove existing demo content (reverse dependency order):**
    First check which demo content plugins are active:
    ```bash
    ddev drush php:eval "
      \$manager = \Drupal::service('plugin.manager.demo_content');
      \$defs = \$manager->getDefinitions();
      foreach (['social_demo_' => 'social_demo', 'localnodes_' => 'localnodes', 'boulder_' => 'boulder'] as \$prefix => \$label) {
        \$count = count(array_filter(\$defs, fn(\$d) => str_starts_with(\$d['id'], \$prefix)));
        echo \"\$label: \$count plugins\" . PHP_EOL;
      }
    "
    ```

    Then remove content in reverse dependency order. Remove ALL demo content that might be present:
    ```bash
    # Remove localnodes_demo content (if present)
    ddev drush sdr localnodes_like localnodes_event_enrollment localnodes_comment localnodes_post localnodes_event localnodes_topic localnodes_group localnodes_user localnodes_file localnodes_event_type localnodes_user_terms 2>/dev/null || true

    # Remove social_demo content (if present)
    ddev drush sdr like event_enrollment comment post event topic group user file user_terms 2>/dev/null || true
    ```

    **Step 2: Verify demo content is removed:**
    ```bash
    ddev drush php:eval "
      \$posts = \Drupal::entityTypeManager()->getStorage('post')->loadMultiple();
      \$groups = \Drupal::entityTypeManager()->getStorage('group')->loadMultiple();
      echo 'Posts remaining: ' . count(\$posts) . PHP_EOL;
      echo 'Groups remaining: ' . count(\$groups) . PHP_EOL;
    "
    ```
    Expected: Posts: 0, Groups: 0 (or very low if admin-created content exists).

    **Step 3: Install boulder_demo content in strict dependency order:**
    ```bash
    ddev drush sda boulder_user_terms boulder_file boulder_user boulder_group boulder_event_type boulder_event boulder_topic boulder_post boulder_comment boulder_like boulder_event_enrollment
    ```

    **IMPORTANT:** Watch the drush output carefully for:
    - "already exists" warnings (UUID collision -- should not happen with fresh UUIDs)
    - "doesn't exists" errors (broken cross-reference -- indicates wrong UUID in a reference field)
    - PHP errors (missing required fields, wrong entity type, etc.)

    If errors occur, note the specific entity UUID and field causing the issue. Common fixes:
    - UUID typo: fix the UUID in the YAML file and re-run just that plugin
    - Missing dependency: ensure the referenced entity's plugin was created first
    - Wrong field name: check localnodes_demo's original YAML for correct field structure

    **Step 4: Verify entity counts:**
    ```bash
    ddev drush php:eval "
      echo 'Users: ' . count(\Drupal::entityTypeManager()->getStorage('user')->loadMultiple()) . PHP_EOL;
      echo 'Groups: ' . count(\Drupal::entityTypeManager()->getStorage('group')->loadMultiple()) . PHP_EOL;
      echo 'Topics+Events: ' . count(\Drupal::entityTypeManager()->getStorage('node')->loadMultiple()) . PHP_EOL;
      echo 'Posts: ' . count(\Drupal::entityTypeManager()->getStorage('post')->loadMultiple()) . PHP_EOL;
      echo 'Comments: ' . count(\Drupal::entityTypeManager()->getStorage('comment')->loadMultiple()) . PHP_EOL;
      echo 'Files: ' . count(\Drupal::entityTypeManager()->getStorage('file')->loadMultiple()) . PHP_EOL;
    "
    ```

    Expected counts (minimum):
    - Users: 12+ (plus admin user 1)
    - Groups: 5
    - Topics + Events: 18 (10 topics + 8 events)
    - Posts: 18
    - Comments: 13+
    - Files: 17+

    **Step 5: Verify Research and Question topic types display correctly:**
    ```bash
    ddev drush php:eval "
      \$query = \Drupal::entityTypeManager()->getStorage('node')->getQuery()
        ->condition('type', 'topic')
        ->accessCheck(FALSE);
      \$nids = \$query->execute();
      \$nodes = \Drupal::entityTypeManager()->getStorage('node')->loadMultiple(\$nids);
      foreach (\$nodes as \$node) {
        \$type_field = \$node->get('field_topic_type');
        \$type_name = '';
        if (!\$type_field->isEmpty()) {
          \$term = \$type_field->entity;
          \$type_name = \$term ? \$term->getName() : 'MISSING';
        }
        echo \$node->getTitle() . ' => ' . \$type_name . PHP_EOL;
      }
    "
    ```
    Expected: At least 1 topic shows "Research" type and 1 shows "Question" type.

    **Step 6: Trigger Search API indexing for AI features:**
    ```bash
    ddev drush search-api:index social_posts
    ddev drush search-api:index social_comments
    ```
    This ensures the new content gets indexed into Qdrant for the AI knowledge garden.

  <action>
    Run automated verification checks to confirm content integrity before manual UI review.

    **Check 1: Group membership verification**
    ```bash
    ddev drush php:eval "
      \$groups = \Drupal::entityTypeManager()->getStorage('group')->loadMultiple();
      foreach (\$groups as \$group) {
        \$members = \$group->getMembers();
        echo \$group->label() . ': ' . count(\$members) . ' members' . PHP_EOL;
      }
    "
    ```
    Expected: Each of the 5 groups has 5-8 members.

    **Check 2: Visibility distribution**
    ```bash
    ddev drush php:eval "
      // Check post visibility distribution
      \$posts = \Drupal::entityTypeManager()->getStorage('post')->loadMultiple();
      \$vis = [0 => 0, 1 => 0, 2 => 0, 3 => 0];
      foreach (\$posts as \$post) {
        \$v = (int) \$post->get('field_visibility')->value;
        \$vis[\$v]++;
      }
      echo 'Posts by visibility: recipient=' . \$vis[0] . ' community=' . \$vis[1] . ' public=' . \$vis[2] . ' group=' . \$vis[3] . PHP_EOL;

      // Check topic visibility
      \$topics = \Drupal::entityTypeManager()->getStorage('node')->getQuery()
        ->condition('type', 'topic')
        ->accessCheck(FALSE)
        ->execute();
      \$nodes = \Drupal::entityTypeManager()->getStorage('node')->loadMultiple(\$topics);
      \$topic_vis = [];
      foreach (\$nodes as \$node) {
        \$v = \$node->get('field_content_visibility')->value ?? 'unknown';
        \$topic_vis[\$v] = (\$topic_vis[\$v] ?? 0) + 1;
      }
      echo 'Topics by visibility: ';
      foreach (\$topic_vis as \$k => \$v) echo \"\$k=\$v \";
      echo PHP_EOL;
    "
    ```
    Expected: Posts have mix of all 4 visibility levels. Topics have public, community, and group.

    **Check 3: Comment threading and entity_type distribution**
    ```bash
    ddev drush php:eval "
      \$comments = \Drupal::entityTypeManager()->getStorage('comment')->loadMultiple();
      \$threaded = 0;
      \$by_type = [];
      foreach (\$comments as \$comment) {
        if (\$comment->getParentComment()) \$threaded++;
        \$bundle = \$comment->bundle();
        \$by_type[\$bundle] = (\$by_type[\$bundle] ?? 0) + 1;
      }
      echo 'Total comments: ' . count(\$comments) . PHP_EOL;
      echo 'Threaded replies: ' . \$threaded . PHP_EOL;
      foreach (\$by_type as \$t => \$c) echo \"  \$t: \$c\" . PHP_EOL;
    "
    ```
    Expected: 13+ total comments, 2+ threaded replies, mix of `comment` and `post_comment` bundles.

    **Check 4: Timezone verification**
    ```bash
    ddev drush php:eval "
      \$users = \Drupal::entityTypeManager()->getStorage('user')->loadMultiple();
      \$denver = 0;
      foreach (\$users as \$user) {
        if (\$user->id() > 1 && \$user->getTimeZone() === 'America/Denver') \$denver++;
      }
      echo 'Users with America/Denver timezone: ' . \$denver . PHP_EOL;
    "
    ```
    Expected: 12 users with America/Denver timezone.

    **Check 5: Likes and enrollments exist**
    ```bash
    ddev drush php:eval "
      \$votes = \Drupal::entityTypeManager()->getStorage('vote')->loadMultiple();
      echo 'Likes: ' . count(\$votes) . PHP_EOL;
      \$enrollments = \Drupal::entityTypeManager()->getStorage('event_enrollment')->loadMultiple();
      echo 'Enrollments: ' . count(\$enrollments) . PHP_EOL;
    "
    ```
    Expected: 9 likes, 10 enrollments.

    If any check fails, investigate and fix the YAML file causing the issue, then re-run the failing plugin(s) via `ddev drush sdr boulder_X && ddev drush sda boulder_X`.
