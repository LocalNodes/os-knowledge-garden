- **Root cause:** New Coolify apps don't have compose file parsed yet, so PATCH docker_compose_domains fails with "Cannot set docker_compose_domains without docker_compose_raw"

- **Working fix:** Include docker_compose_domains + docker_compose_location in the CREATE call (POST /applications/public). Coolify handles sequencing internally.
- Removed separate checkout step and domain PATCH step — simpler workflow
