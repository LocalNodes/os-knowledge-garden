  <action>
Run the following verification checks for BOTH cascadia.localnodes.xyz and boulder.localnodes.xyz:

**1. HTTP accessibility:**
```bash
curl -sI https://cascadia.localnodes.xyz | head -5
curl -sI https://boulder.localnodes.xyz | head -5
```
Expect: `HTTP/2 200` (or 301/302 redirect to the site). If 503, the container is still starting — wait and retry.

**2. SSL certificate validation:**
```bash
echo | openssl s_client -connect cascadia.localnodes.xyz:443 -servername cascadia.localnodes.xyz 2>/dev/null | openssl x509 -noout -subject -issuer -dates
echo | openssl s_client -connect boulder.localnodes.xyz:443 -servername boulder.localnodes.xyz 2>/dev/null | openssl x509 -noout -subject -issuer -dates
```
Expect: Valid cert from Let's Encrypt (or Google Trust Services via Cloudflare). Not expired.

**3. Correct site name:**
```bash
curl -s https://cascadia.localnodes.xyz/user/login | grep -o '<title>[^<]*</title>'
curl -s https://boulder.localnodes.xyz/user/login | grep -o '<title>[^<]*</title>'
```
Expect: Cascadia shows "Cascadia LocalNodes" in the title. Boulder shows "Boulder LocalNodes".

**4. Container health via Coolify:**
```
mcp__coolify__diagnose_app(query="cascadia")
mcp__coolify__diagnose_app(query="boulder")
```
Expect: All 4 containers per instance show `running:healthy`.

**5. Deployment logs check (if any issues):**
```
mcp__coolify__application_logs(uuid="<cascadia-uuid>", lines=50)
mcp__coolify__application_logs(uuid="<boulder-uuid>", lines=50)
```
Look for: "FRESH INSTALL COMPLETE" or "EXISTING INSTALL READY" — confirms entrypoint ran successfully.

If any check fails, document the failure and check deployment logs for error details.
