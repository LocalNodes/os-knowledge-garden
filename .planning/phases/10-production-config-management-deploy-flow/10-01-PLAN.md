  <action>
Replace the existing 903-line stock settings.php (which is the Drupal default with a DDEV include and config_sync_directory appended at bottom) with a clean template that is purpose-built for this project. The new settings.php will be committed to git and used in both DDEV (local) and Docker (production) environments.

The new settings.php MUST include (in this order):

1. **Database configuration via getenv():**
```php
$databases['default']['default'] = [
  'database' => getenv('DB_NAME') ?: 'opensocial',
  'username' => getenv('DB_USER') ?: 'opensocial',
  'password' => getenv('DB_PASSWORD') ?: 'changeme',
  'host' => getenv('DB_HOST') ?: 'mariadb',
  'port' => getenv('DB_PORT') ?: '3306',
  'driver' => 'mysql',
  'prefix' => '',
];
```

2. **Core settings:**
```php
$settings['hash_salt'] = getenv('DRUPAL_HASH_SALT') ?: 'change-me-in-production';
$settings['config_sync_directory'] = '../config/sync';
$settings['file_private_path'] = '/var/www/private';
$settings['update_free_access'] = FALSE;
$settings['container_yamls'][] = $app_root . '/' . $site_path . '/services.yml';
```

3. **config_exclude_modules** — Exclude modules that differ per instance:
```php
$settings['config_exclude_modules'] = [
  'localnodes_demo',
  'boulder_demo',
  'portland_demo',
  'social_demo',
  'siwe_login',
  'safe_smart_accounts',
  'group_treasury',
  'social_group_treasury',
];
```

4. **Solr config overrides** (runtime layer, not stored in DB):
```php
$config['search_api.server.social_solr']['backend_config']['connector_config']['host'] = getenv('SOLR_HOST') ?: 'solr';
$config['search_api.server.social_solr']['backend_config']['connector_config']['port'] = getenv('SOLR_PORT') ?: '8983';
$config['search_api.server.social_solr']['backend_config']['connector_config']['core'] = 'drupal';
$config['search_api.server.social_solr']['backend_config']['connector_config']['path'] = '/';
```

5. **Qdrant config overrides:**
```php
$config['ai_vdb_provider_qdrant.settings']['host'] = getenv('QDRANT_HOST') ?: 'qdrant';
$config['ai_vdb_provider_qdrant.settings']['port'] = (int)(getenv('QDRANT_PORT') ?: '6333');
```

6. **Gemini API key via environment:**
```php
$config['key.key.gemini_api_key']['key_provider_settings']['env_variable'] = 'GEMINI_API_KEY';
```

7. **Reverse proxy support** (Coolify/Traefik):
```php
if (getenv('DRUPAL_REVERSE_PROXY') === 'true') {
  $settings['reverse_proxy'] = TRUE;
  $settings['reverse_proxy_addresses'] = ['0.0.0.0/0'];
  $settings['reverse_proxy_trusted_headers'] =
    \Symfony\Component\HttpFoundation\Request::HEADER_X_FORWARDED_FOR |
    \Symfony\Component\HttpFoundation\Request::HEADER_X_FORWARDED_HOST |
    \Symfony\Component\HttpFoundation\Request::HEADER_X_FORWARDED_PORT |
    \Symfony\Component\HttpFoundation\Request::HEADER_X_FORWARDED_PROTO;
}
```

8. **Trusted host patterns from env:**
```php
$fqdn = getenv('SERVICE_FQDN_OPENSOCIAL');
if ($fqdn) {
  $host = preg_replace('#^https?://#', '', $fqdn);
  $settings['trusted_host_patterns'][] = '^' . preg_quote($host, '/') . '$';
}
$settings['trusted_host_patterns'][] = '^localhost$';
```

9. **DDEV integration** (at the bottom, so DDEV overrides take precedence):
```php
if (getenv('IS_DDEV_PROJECT') == 'true' && file_exists(__DIR__ . '/settings.ddev.php')) {
  include __DIR__ . '/settings.ddev.php';
}
```

- Do NOT include the 850+ lines of commented-out Drupal default documentation. This is a purpose-built template, not a stock file.

  <name>Task 2: Add config/sync to Docker image and create deploy hook scaffold</name>
  <files>docker/Dockerfile, html/modules/custom/localnodes_platform/localnodes_platform.deploy.php</files>
  <action>
**Dockerfile changes:**

In `docker/Dockerfile`, add a COPY line in the runtime stage to include the config/sync directory. Add it AFTER the existing COPY lines (after `COPY html/modules/custom/`) and BEFORE the `RUN mkdir -p` lines:

```dockerfile
# Copy config sync directory (for drush deploy / drush cim)
COPY config/ /var/www/html/config/
```

This places config/sync at `/var/www/html/config/sync/` inside the container, which matches the `$settings['config_sync_directory'] = '../config/sync'` path (relative to `/var/www/html/html/sites/default/`, that resolves to `/var/www/html/config/sync/`).

Also, since we are shipping a template settings.php committed to git, the entrypoint no longer needs to generate it. But the Dockerfile's COPY of `html/` from the builder already includes `html/sites/default/settings.php`. Verify this is the case — the template settings.php will be included automatically.

**Deploy hook file:**

Create `html/modules/custom/localnodes_platform/localnodes_platform.deploy.php` with the following content:

```php
<?php

/**
 * @file
 * Deploy hooks for localnodes_platform.
 *
 * Deploy hooks run AFTER config import during `drush deploy`.
 * Use for one-time content/state changes that depend on new config.
 *
 * Naming convention: localnodes_platform_deploy_NNNN_description()
 *
 * @see https://www.drush.org/12.x/deploycommand/
 */
```

This is intentionally empty (just the docblock). It establishes the file location and naming convention for when deploy hooks are needed in the future.
