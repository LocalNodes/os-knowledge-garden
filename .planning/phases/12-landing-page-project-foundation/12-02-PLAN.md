  <name>Task 1: Configure Resend DNS records for localnodes.xyz email</name>
  <files>N/A - DNS configuration in external dashboards</files>
  <action>
    This requires access to two dashboards that Claude cannot access: the Resend dashboard and the Cloudflare dashboard.

    **Step 1: Add domain in Resend**
    1. Go to https://resend.com/domains
    2. Click "Add Domain"
    3. Enter `localnodes.xyz`
    4. Choose a sending subdomain (Resend typically suggests `send.localnodes.xyz` to avoid MX record conflicts)
    5. Resend will generate 3-5 DNS records to add

    **Step 2: Add DNS records in Cloudflare**
    1. Go to Cloudflare dashboard, localnodes.xyz, DNS, Records
    2. Add each record Resend provided:
       - 2x CNAME records for DKIM (e.g., `resend._domainkey.localnodes.xyz`)
       - 1x TXT record for SPF (e.g., `send.localnodes.xyz` with value `v=spf1 include:amazonses.com ~all`)
       - Any MX record if Resend provides one
    3. CRITICAL: Set proxy status to "DNS Only" (grey cloud) for ALL CNAME records -- Cloudflare proxy breaks DKIM validation
    4. Add DMARC TXT record at `_dmarc.localnodes.xyz` with value: `v=DMARC1; p=none; rua=mailto:dmarc@localnodes.xyz;`
       - `p=none` is intentional -- monitor only during warmup. Upgrade to `p=quarantine` after Phase 16 confirms emails work.

    **Alternative: Automatic setup**
    - If Resend shows a "Sign in to Cloudflare" button (Domain Connect), use that for one-click setup
    - You will still need to manually add the DMARC record

    **Step 3: Verify in Resend**
    1. Click "Verify DNS Records" in Resend's domain settings
    2. DNS propagation may take 15 minutes to 72 hours
    3. If verification succeeds, the domain status shows "Verified"
    4. If stuck in "Pending" after 24 hours, double-check that CNAME proxy is set to "DNS Only"

    Resume signal: Confirm DNS records have been added and Resend domain status (Pending/Verified). Type "dns configured" or describe any issues.
  </action>
