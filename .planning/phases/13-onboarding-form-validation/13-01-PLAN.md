  <name>Task 2: Create check-subdomain server route with runtimeConfig and tests</name>
  <files>
    localnodes-onboarding/nuxt.config.ts,

    localnodes-onboarding/tests/unit/check-subdomain.test.ts,

  </files>
  <behavior>
    - Server route validates slug query parameter (min 3, max 63, regex for valid DNS label)
    - Reserved subdomains (www, api, coolify, mail, smtp, admin, app, dashboard, status, billing, support, help, docs) return { available: false, reason: 'This name is reserved' }
    - Valid non-reserved slug queries Coolify API for existing applications
    - Taken subdomain returns { available: false, reason: 'This subdomain is already in use' }
    - Available subdomain returns { available: true, reason: null }
    - Invalid slug (too short, special chars, starts with hyphen) returns 400 error
    - Coolify API failure returns 502 error
  </behavior>
  <action>
    Working directory: localnodes-onboarding/

    1. Update `nuxt.config.ts` -- add runtimeConfig block with private Coolify API credentials and add `/onboarding` route rule:
       ```typescript
       runtimeConfig: {
         coolifyApiUrl: 'https://coolify.localnodes.xyz/api/v1',
         coolifyApiToken: '',  // Set via NUXT_COOLIFY_API_TOKEN env var
       },
       routeRules: {
         '/': { prerender: true },
         '/onboarding': { ssr: true }
       }
       ```
       Keep all existing config (modules, css, colorMode, app, compatibilityDate, devtools).

    2. Create `server/api/check-subdomain.get.ts`:
       - Import `* as v from 'valibot'` for query validation
       - Define querySchema: slug must be string, minLength(3), maxLength(63), regex(/^[a-z0-9]([a-z0-9-]*[a-z0-9])?$/)
       - Define RESERVED_SUBDOMAINS constant array
       - Use `getValidatedQuery` with Valibot parse for type-safe validation
       - If slug is reserved, return `{ available: false, reason: 'This name is reserved' }`
       - Use `useRuntimeConfig(event)` to get coolifyApiUrl and coolifyApiToken
       - Fetch `${coolifyApiUrl}/applications` with Bearer token
       - Check if any app's `docker_compose_domains` contains `${slug}.localnodes.xyz`
       - Handle `docker_compose_domains` as either string or object (use JSON.stringify for both)
       - Return `{ available: boolean, reason: string | null }`
       - On Coolify API error, throw createError with status 502

    3. Create `tests/unit/check-subdomain.test.ts`:
       - Test the reserved subdomain logic as a pure function (extract RESERVED_SUBDOMAINS check)
       - Test query parameter validation (slug format requirements)
       - For Coolify API integration, test that the domain matching logic works correctly:
         mock an apps array and verify the `taken` check logic
       - Note: Full integration tests require the Nuxt dev server running. Unit tests focus on the
         validation logic and domain matching algorithm.

    4. Create `.env.example` documenting required env vars:
       ```
       # Coolify API (required for subdomain availability checks)
       NUXT_COOLIFY_API_TOKEN=your_coolify_api_token_here
       ```

    5. Verify: Run all tests pass.
