<task type="auto">
  <name>Task 1: Create useSubdomain composable and SubdomainPreview component</name>
  <files>
    localnodes-onboarding/app/composables/useSubdomain.ts,
    localnodes-onboarding/app/components/SubdomainPreview.vue
  </files>
  <action>
    Working directory: localnodes-onboarding/

    1. Create `app/composables/useSubdomain.ts`:
       - Accept `communityName: Ref<string>` parameter
       - Import `slugify` from `~/utils/slugify` (Nuxt auto-imports utils, but explicit import is clearer)
       - Import `refDebounced` from `@vueuse/core`
       - Create `slug` as computed from `slugify(communityName.value)`
       - Create `debouncedSlug` as `refDebounced(slug, 500)` (500ms debounce)
       - Track `availability` as ref with type union: 'idle' | 'checking' | 'available' | 'taken' | 'invalid'
       - Track `errorMessage` as `ref<string | null>(null)`
       - Watch `debouncedSlug`: if empty or length < 3, set availability to 'idle' and return.
         Otherwise set 'checking', call `$fetch('/api/check-subdomain', { query: { slug } })`,
         update availability to 'available' or 'taken' based on response, set errorMessage.
         On catch, set 'invalid' and 'Could not check availability'.
       - Watch raw `slug`: when it differs from debouncedSlug, set availability to 'checking'
         (immediate feedback during debounce delay). Do NOT use immediate: true on this watcher.
       - Create `subdomain` computed: `slug.value ? \`${slug.value}.localnodes.xyz\` : ''`
       - Return `{ slug, subdomain, availability, errorMessage }`

    2. Create `app/components/SubdomainPreview.vue`:
       - Accept props: `subdomain: string`, `availability: 'idle' | 'checking' | 'available' | 'taken' | 'invalid'`
       - Only render when `subdomain` is truthy (use `v-if="subdomain"`)
       - Layout: flex row with icon, monospace subdomain text, and status text
       - Use `UIcon` with computed icon name based on availability:
         - 'checking': 'i-lucide-loader-circle' (add `animate-spin` class)
         - 'available': 'i-lucide-check-circle'
         - 'taken': 'i-lucide-x-circle'
         - 'invalid': 'i-lucide-alert-circle'
         - 'idle': 'i-lucide-globe'
       - Use computed color class based on availability:
         - 'available': 'text-green-500'
         - 'taken': 'text-red-500'
         - 'invalid': 'text-amber-500'
         - default: 'text-(--ui-text-muted)'
       - Status text: 'Checking...', 'Available!', 'Already taken', 'Could not check', '' (idle)
  </action>

  </verify>
  <done>useSubdomain composable provides reactive slug, debounced availability checking with 5 states, and subdomain string. SubdomainPreview component renders the visual indicator with correct icons and colors for each state.</done>
</task>

<task type="auto">
  <name>Task 2: Create OnboardingForm component and onboarding page</name>
  <files>
    localnodes-onboarding/app/components/OnboardingForm.vue,
    localnodes-onboarding/app/pages/onboarding.vue
  </files>
  <action>
    Working directory: localnodes-onboarding/

    1. Create `app/components/OnboardingForm.vue`:
       - Import `* as v from 'valibot'` for schema definition
       - Define Valibot schema:
         ```
         communityName: v.pipe(v.string(), v.minLength(3, 'Community name must be at least 3 characters'), v.maxLength(50, 'Community name must be under 50 characters'))
         email: v.pipe(v.string(), v.email('Please enter a valid email address'))
         password: v.pipe(v.string(), v.minLength(8, 'Password must be at least 8 characters'))
         ```
       - Infer `Schema` type with `v.InferOutput<typeof schema>`
       - Create reactive `state` object with communityName, email, password (all empty strings)
       - Call `useSubdomain(toRef(() => state.communityName))` to get slug, subdomain, availability, errorMessage
       - Create `showPassword` ref (false)
       - Create `canSubmit` computed: `availability.value === 'available'`
       - Create `onSubmit` handler: for Phase 13, log the data and navigate to a placeholder.
         Use `navigateTo('/onboarding/confirm')` (Phase 14 will replace with Stripe redirect).
         Guard: if availability !== 'available', return early (defense in depth).
       - Template structure using UForm with `:schema` and `:state` props, `@submit="onSubmit"`:

         a. Community Name field: `UFormField` (label="Community Name", name="communityName", required)
            with `UInput` (v-model="state.communityName", placeholder="e.g., Cascadia Bioregion").
            Below the UFormField, render `<SubdomainPreview>` with subdomain and availability props.
            If `errorMessage` from useSubdomain, show it as additional hint text in red.

         b. Email field: `UFormField` (label="Email Address", name="email", required)
            with `UInput` (v-model="state.email", type="email", placeholder="you@example.com")

         c. Password field: `UFormField` (label="Password", name="password", required)
            with `UInput` (v-model="state.password", :type="showPassword ? 'text' : 'password'",
            placeholder="At least 8 characters"). Use trailing slot with UButton
            (icon toggle: i-lucide-eye / i-lucide-eye-off, variant="ghost", size="xs",
            :padded="false", @click to toggle showPassword).

         d. Submit button: `UButton` (type="submit", label="Continue to Payment",
            block, size="xl", :disabled="!canSubmit || availability === 'checking'",
            :loading="availability === 'checking'").
            Below the button, add a small "You'll be charged $29/month" note in muted text.

       - Set `validate-on` prop on UForm to `['blur', 'submit']` so users get feedback when
         tabbing between fields, not just on submit.

    2. Create `app/pages/onboarding.vue`:
       - Use `UPageSection` wrapper (consistent with landing page pattern)
       - Centered layout: `<div class="max-w-md mx-auto">`
       - Heading: "Name your community" (h1, text-2xl font-bold)
       - Subheading: "Choose a name for your knowledge garden. This will become your unique subdomain."
         (text-muted, mb-8)
       - Render `<OnboardingForm />`
       - Add SEO meta: title "Get Started - LocalNodes", description "Set up your community knowledge garden in minutes."
       - Add `defineRouteRules({ ssr: true })` (matches nuxt.config.ts routeRules)

    3. Verify the form renders by running the dev server and checking the page loads without errors.
