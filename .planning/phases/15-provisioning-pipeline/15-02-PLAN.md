  <action>
    Modify the existing `.github/workflows/provision-instance.yml` to add post-provisioning automation. All changes are additive -- existing steps are preserved, new steps are inserted after them.

    1. Add new workflow_dispatch inputs (after existing `demo_module` input):
       ```yaml
       email:
         description: 'Organizer email address'
         required: false
         type: string
       stripe_session_id:
         description: 'Stripe Checkout Session ID for state tracking'
         required: false
         type: string
       callback_url:
         description: 'Base URL to POST status updates to (e.g., https://localnodes.xyz)'
         required: false
         type: string
       ```
       NOTE: All new inputs are `required: false` so the workflow remains backward-compatible with manual dispatch (which doesn't provide email/session_id).

    2. Add a reusable "Update provisioning status" step as a shell function (or repeat as needed). Insert a status callback step after each major phase:

       **After "Deploy instance" step** -- add callback with status `provisioning`:
       ```yaml
       - name: Notify status - provisioning
         if: inputs.callback_url != '' && inputs.stripe_session_id != ''
         run: |
           curl -sf -X POST "${{ inputs.callback_url }}/api/provision-callback" \
             -H "Content-Type: application/json" \
             -H "Authorization: Bearer ${{ secrets.PROVISION_CALLBACK_SECRET }}" \
             -d "{
               \"session_id\": \"${{ inputs.stripe_session_id }}\",
               \"status\": \"provisioning\",
               \"site_url\": \"https://$FQDN\"
             }" || echo "Warning: callback failed (non-fatal)"
       ```

       **After "Wait for deployment" step** -- add callback with status `installing`:
       ```yaml
       - name: Notify status - installing
         if: inputs.callback_url != '' && inputs.stripe_session_id != ''
         run: |
           curl -sf -X POST "${{ inputs.callback_url }}/api/provision-callback" \
             -H "Content-Type: application/json" \
             -H "Authorization: Bearer ${{ secrets.PROVISION_CALLBACK_SECRET }}" \
             -d "{
               \"session_id\": \"${{ inputs.stripe_session_id }}\",
               \"status\": \"installing\",
               \"site_url\": \"https://$FQDN\"
             }" || echo "Warning: callback failed (non-fatal)"
       ```

    3. After "Wait for site to become healthy" step, store HTTP_CODE in GITHUB_ENV:
       Add `echo "HTTP_CODE=$HTTP_CODE" >> "$GITHUB_ENV"` at the end of that step (before the warning check).

    4. **Use `webfactory/ssh-agent@v0.9.0` for SSH key injection** so LOGIN_URL can be captured in GITHUB_ENV (appleboy/ssh-action cannot export env vars back to the workflow):

       ```yaml
       - name: Setup SSH
         if: env.HTTP_CODE == '200' && inputs.email != ''
         uses: webfactory/ssh-agent@v0.9.0
         with:
           ssh-private-key: ${{ secrets.SSH_PRIVATE_KEY }}
       ```

    5. Add "Notify status - creating_user" callback BEFORE the user creation step:
       ```yaml
       - name: Notify status - creating user
         if: env.HTTP_CODE == '200' && inputs.email != '' && inputs.callback_url != '' && inputs.stripe_session_id != ''
         run: |
           curl -sf -X POST "${{ inputs.callback_url }}/api/provision-callback" \
             -H "Content-Type: application/json" \
             -H "Authorization: Bearer ${{ secrets.PROVISION_CALLBACK_SECRET }}" \
             -d "{
               \"session_id\": \"${{ inputs.stripe_session_id }}\",
               \"status\": \"creating_user\",
               \"site_url\": \"https://$FQDN\"
             }" || echo "Warning: callback failed (non-fatal)"
       ```

    6. Add "Create organizer admin account" step:
       ```yaml
       - name: Create organizer admin account
         if: env.HTTP_CODE == '200' && inputs.email != ''
         run: |
           # Add host key to known_hosts to avoid interactive prompt
           ssh-keyscan -H localnodes.xyz >> ~/.ssh/known_hosts 2>/dev/null

           # Find the opensocial container
           CONTAINER=$(ssh root@localnodes.xyz "docker ps --filter 'name=${{ inputs.subdomain }}' --filter 'name=opensocial' --format '{{.Names}}' | head -1")
           if [ -z "$CONTAINER" ]; then
             echo "::error::Could not find opensocial container for ${{ inputs.subdomain }}"
             exit 1
           fi
           echo "Found container: $CONTAINER"

           DRUSH="docker exec $CONTAINER /var/www/html/vendor/bin/drush -r /var/www/html/html --uri=https://$FQDN"

           # Generate secure random password
           PASSWORD=$(openssl rand -hex 16)

           # Create user with community name + " Admin" as username
           USERNAME="${{ inputs.site_name }} Admin"
           ssh root@localnodes.xyz "$DRUSH user:create \"$USERNAME\" --mail=\"${{ inputs.email }}\" --password=\"$PASSWORD\"" 2>&1 || echo "Warning: user may already exist"

           # Assign sitemanager role
           ssh root@localnodes.xyz "$DRUSH user:role:add sitemanager \"$USERNAME\"" 2>&1 || echo "Warning: role may already be assigned"

           # Generate one-time login URL
           LOGIN_URL=$(ssh root@localnodes.xyz "$DRUSH uli --name=\"$USERNAME\" --uri=https://$FQDN" 2>/dev/null)
           echo "LOGIN_URL=$LOGIN_URL" >> "$GITHUB_ENV"
           echo "Login URL generated for ${{ inputs.email }}"
       ```

    7. Add "Send welcome email" step after user creation:
       ```yaml
       - name: Send welcome email
         if: env.LOGIN_URL != '' && inputs.email != ''
         run: |
           # Notify status - sending email
           if [ -n "${{ inputs.callback_url }}" ] && [ -n "${{ inputs.stripe_session_id }}" ]; then
             curl -sf -X POST "${{ inputs.callback_url }}/api/provision-callback" \
               -H "Content-Type: application/json" \
               -H "Authorization: Bearer ${{ secrets.PROVISION_CALLBACK_SECRET }}" \
               -d "{
                 \"session_id\": \"${{ inputs.stripe_session_id }}\",
                 \"status\": \"sending_email\",
                 \"site_url\": \"https://$FQDN\",
                 \"login_url\": \"$LOGIN_URL\"
               }" || echo "Warning: callback failed (non-fatal)"
           fi

           # Send welcome email via Resend API
           RESPONSE=$(curl -s -w "\n%{http_code}" -X POST "https://api.resend.com/emails" \
             -H "Authorization: Bearer ${{ secrets.RESEND_API_KEY }}" \
             -H "Content-Type: application/json" \
             -d "{
               \"from\": \"LocalNodes <hello@localnodes.xyz>\",
               \"to\": [\"${{ inputs.email }}\"],
               \"subject\": \"Your knowledge garden is ready!\",
               \"html\": \"<h1>Welcome to LocalNodes!</h1><p>Your knowledge garden at <a href='https://$FQDN'>https://$FQDN</a> is ready.</p><p><a href='$LOGIN_URL'>Click here to set your password and log in</a></p><p>This login link is single-use and will expire after first use. If it expires, you can request a new one from the login page.</p><p>Happy gardening!<br>The LocalNodes Team</p>\"
             }")

           HTTP_CODE=$(echo "$RESPONSE" | tail -1)
           BODY=$(echo "$RESPONSE" | head -n -1)
           if [ "$HTTP_CODE" = "200" ]; then
             echo "Welcome email sent to ${{ inputs.email }}"
           else
             echo "::warning::Welcome email failed (HTTP $HTTP_CODE): $BODY"
           fi
       ```

    8. Add "Notify status - complete" step at the end (after welcome email, before Summary):
       ```yaml
       - name: Notify status - complete
         if: env.HTTP_CODE == '200' && inputs.callback_url != '' && inputs.stripe_session_id != ''
         run: |
           curl -sf -X POST "${{ inputs.callback_url }}/api/provision-callback" \
             -H "Content-Type: application/json" \
             -H "Authorization: Bearer ${{ secrets.PROVISION_CALLBACK_SECRET }}" \
             -d "{
               \"session_id\": \"${{ inputs.stripe_session_id }}\",
               \"status\": \"complete\",
               \"site_url\": \"https://$FQDN\",
               \"login_url\": \"${LOGIN_URL:-}\"
             }" || echo "Warning: callback failed (non-fatal)"
       ```

    9. Add failure callback at the very end of the job:
       ```yaml
       - name: Notify status - failed
         if: failure() && inputs.callback_url != '' && inputs.stripe_session_id != ''
         run: |
           curl -sf -X POST "${{ inputs.callback_url }}/api/provision-callback" \
             -H "Content-Type: application/json" \
             -H "Authorization: Bearer ${{ secrets.PROVISION_CALLBACK_SECRET }}" \
             -d "{
               \"session_id\": \"${{ inputs.stripe_session_id }}\",
               \"status\": \"failed\",
               \"error\": \"Provisioning workflow failed. Check GitHub Actions logs for details.\"
             }" || echo "Warning: callback failed (non-fatal)"
       ```

    10. Update the "Summary" step to include organizer account info:
        Add to the summary table:
        ```
        | **Organizer Email** | ${{ inputs.email }} |
        | **Login URL** | ${{ env.LOGIN_URL }} |
        ```

    IMPORTANT IMPLEMENTATION NOTES:
    - All callback curl commands use `|| echo "Warning: ..."` so callback failures never block provisioning
    - All new steps have `if:` conditions so existing manual-dispatch usage (without email/session_id) is unaffected
    - The `webfactory/ssh-agent` action handles SSH key injection and cleanup automatically
    - User creation commands use `|| echo "Warning: ..."` for idempotency (PROV-04) -- if user already exists from a previous attempt, the step doesn't fail
    - drush uli generates a URL with a one-time token that expires on first use -- Drupal handles this natively
